## H_A: SOAP Optimizer (Plateau Protocol — Tier 1 Priority)

### Hypothesis
Replace Lion with SOAP (Shampoo + Adam In Preconditioner's Eigenbasis, NeurIPS 2024). SOAP
computes full matrix preconditioners via Shampoo but applies updates in the preconditioner's
eigenbasis, giving second-order curvature geometry at Adam's cost. Lion's sign-based update
discards gradient magnitude — critical for τ_z where the signal-to-noise ratio is worst. SOAP
restores magnitude information and applies curvature-aware scaling that should selectively
improve high-variance τ_z without amplifying cross-axis noise.

### Why This Addresses the Current Bottleneck
H4 definitively showed Lion sign-update + per-axis weighting amplifies noise on τ_z.
H8 (AdamW lr=5e-4, nezuko PR #1144) is the standard Plateau Protocol response, but SOAP
goes further: AdamW uses diagonal first+second moment (Adam), while SOAP uses full matrix
preconditioners. For a 6-layer transformer with block-diagonal Hessian structure (one block
per attention head), SOAP captures inter-parameter curvature that AdamW misses. This is
particularly relevant for WSS where τ_x / τ_y / τ_z share decoder parameters and
off-diagonal Hessian terms couple their gradients.

### Key Papers
- **SOAP: Improving and Stabilizing Shampoo using Adam** (Vyas et al., NeurIPS 2024):
  https://arxiv.org/abs/2409.11321 — Matching or exceeding AdamW on LLM pretraining at same
  wall-clock cost; achieves Adam memory overhead while matching Shampoo convergence.
- **Shampoo** (Gupta et al., 2018): https://arxiv.org/abs/1802.09568 — Original matrix
  preconditioner foundation.

### Implementation

```python
# pip install soap-optimizer (or copy from github.com/nikhilvyas/SOAP)
from soap import SOAP

optimizer = SOAP(
    model.parameters(),
    lr=3e-4,                   # Start at 3x lower than H8 AdamW (5e-4), SOAP effective LR is higher
    betas=(0.95, 0.95),        # Recommended starting point from paper
    shampoo_beta=-1,           # -1 uses the outer beta2 for Shampoo EMA
    eps=1e-8,
    weight_decay=1e-2,
    precondition_frequency=10, # Recompute eigenbasis every 10 steps (cheap)
    max_precond_dim=2048,      # Cap preconditioner size for large attention matrices
    merge_dims=False,          # Keep tensor structure; do not flatten
    precondition_1d=True,      # Also precondition 1D params (biases, norms)
    normalize_grads=False,     # Do not normalize; let Shampoo handle scaling
    correct_bias=True,         # Adam-style bias correction
)
```

## H_B: Coarse-to-Fine Point Budget Curriculum

### Hypothesis
Train with an increasing point budget curriculum: 16k surface + 16k volume (EP0–9) →
32k surface + 32k volume (EP10–19) → 65k surface + 65k volume (EP20–29). At each transition,
reset the learning rate to 70% of its initial value (cosine restart). This forces the model to
first learn global aerodynamic structure (front stagnation, wake pattern) at coarse resolution,
then progressively refine to the high-frequency τ_z surface detail that drives WSS error.

### Why This Addresses the Current Bottleneck
The val_wss plateau at ~6.8–7.0% across H5 and H6 is consistent with the model converging
to a local basin that correctly captures global WSS patterns but fails on high-frequency
τ_z variation. Coarse-to-fine curriculum training (arXiv:2509.13138) explicitly demonstrates
"breaking through plateaus" in CFD GNNs by decoupling global structure learning (low-freq)
from fine-scale refinement (high-freq). At 16k points, the model cannot overfit to noisy
τ_z samples; at 65k, it already has a strong prior from coarser stages.

### Key Papers
- **Curriculum Learning for CFD with Graph Neural Networks** (arXiv:2509.13138, 2025):
  32k→65k→131k point budget; LR reset at transitions; 50% wall-clock reduction; directly
  demonstrates plateau-breaking behavior on vehicle aerodynamics GNNs.
- **Curriculum Learning** (Bengio et al., ICML 2009): Original curriculum formulation.
- **PointNet++** multi-resolution architecture: shows coarse→fine hierarchy benefits for
  point clouds.

### Implementation

The current stack already supports variable point counts via `--n_surf_points` and
`--n_vol_points`. The student needs to implement a multi-phase training loop:

```python

    {"surf": 65000, "vol": 65000, "epochs": 10, "lr_scale": 0.49},  # 0.7^2

for phase_idx, phase in enumerate(PHASES):
    if phase_idx > 0:
        # LR cosine restart: reset optimizer state, keep model weights
        for pg in optimizer.param_groups:
            pg['lr'] = base_lr * phase['lr_scale']
    train_phase(phase['surf'], phase['vol'], phase['epochs'])
```

Simplest implementation: run 3 sequential training jobs, each restoring from previous
checkpoint and setting new point budgets + LR. No new dependencies.

## H_C: GATr-Style E(3)-Equivariant Surface Decoder Layer

### Hypothesis
Replace the final surface decoder MLP with a single E(3)-equivariant geometric algebra
attention layer (GATr-style) that maps multivector surface features → (τ_x, τ_y, τ_z)
respecting the pseudo-vector transformation law of wall shear stress under rotation and
reflection. WSS is a pseudo-vector (axial vector): it transforms as r × F, not as a regular
vector. Standard MLP decoders ignore this — they can learn the τ values in one orientation
but generalize poorly to untrained orientations (e.g., varying geometry rotations in the
dataset). A single equivariant layer enforces this symmetry algebraically.

### Why This Addresses the Current Bottleneck
τ_z (8.747% test) is the cross-stream shear component. Its high error relative to τ_x (5.971%)
and τ_y (7.362%) is consistent with the model under-representing the geometric coupling between
the vehicle yaw plane and the near-wall flow direction. An equivariant layer explicitly encodes
how WSS transforms under E(3) — the τ_z prediction is no longer an independent regression but
is geometrically coupled to τ_x and τ_y via the multivector algebra.

GATr (NeurIPS 2023) demonstrated on arterial mesh WSS prediction: 40% reduction in L2 error
vs non-equivariant baseline, 15% vs other equivariant baselines. The vehicle aerodynamics
setting is analogous (3D surface mesh, pseudo-vector output, geometric symmetries).

### Key Papers
- **GATr: Geometric Algebra Transformers** (Brehmer et al., NeurIPS 2023):
  https://arxiv.org/abs/2305.18415 — E(3)-equivariant transformer via geometric algebra
  multivectors; demonstrated on WSS in arterial flow meshes.
- **Equivariant architectures for learning in atomistic simulations** (Batzner et al., 2022):
  Related equivariance for 3D scientific ML.
- **gatr-pytorch** (github.com/geometric-algebra-transformers/gatr): Reference implementation.

### Implementation

The minimal implementation replaces only the WSS output head (not the backbone):

```python
# Install: pip install gatr
from gatr import GATr, SelfAttentionConfig, MLPConfig
from gatr.interface import embed_point, extract_scalar, extract_bivector

class EquivariantWSSHead(nn.Module):
    """Single GATr layer mapping surface point features → WSS pseudo-vector."""
    def __init__(self, in_channels: int, hidden_mv: int = 16, hidden_scalar: int = 64):
        super().__init__()
        # Embed surface features as multivectors (grade-1 vectors + grade-0 scalars)
        self.feat_proj = nn.Linear(in_channels, hidden_mv * 16 + hidden_scalar)
        self.gatr = GATr(
            in_mv_channels=hidden_mv,
            out_mv_channels=hidden_mv,
            hidden_mv_channels=hidden_mv,
            in_s_channels=hidden_scalar,
            out_s_channels=hidden_scalar,
            hidden_s_channels=hidden_scalar,
            num_blocks=1,
            attention=SelfAttentionConfig(num_heads=4),
            mlp=MLPConfig(),
        )
        # Output: extract grade-2 bivector (pseudo-vector = dual of bivector in GA)
        self.out_proj = nn.Linear(hidden_mv * 16 + hidden_scalar, 3)

    def forward(self, x, ref_mv=None):
        # x: (B, N_surf, in_channels)
        proj = self.feat_proj(x)
        mv = proj[..., :self.hidden_mv*16].view(*x.shape[:-1], self.hidden_mv, 16)
        s = proj[..., self.hidden_mv*16:]
        mv_out, s_out = self.gatr(mv, s, reference_mv=ref_mv)
        out = self.out_proj(torch.cat([mv_out.flatten(-2), s_out], dim=-1))
        return out  # (B, N_surf, 3) = (τ_x, τ_y, τ_z)
```

Key: Use `reference_mv` from the global wind direction vector (freestream direction) to
set the geometric frame for equivariance. The wind direction is already available in the
dataset.

### Expected Impact
- WSS improvement: 0.4–1.0pp (τ_z primary; GATr NeurIPS 2023 showed 40% L2 reduction on
  arterial WSS, which is a geometrically similar problem)
- vol_p / surf_p: Unaffected (decoder-only change; backbone unchanged)
- Risk: GATr multivector embedding may not align with Transolver's intermediate feature
  space without tuning. The projection layer bridges this, but may require LR warmup.

### Implementation Difficulty: Medium-High
GATr package is available (`pip install gatr`). Challenge is correctly embedding
Transolver's surface feature vectors as GA multivectors. Student should start with
the simplest embedding (grade-1 = coordinate vector, grade-0 = scalar features) and
verify forward pass produces finite output before training.

Critical check: the pseudo-vector transformation test. WSS under 180-degree rotation
around z-axis should: τ_x → −τ_x, τ_y → −τ_y, τ_z → +τ_z (pseudo-vector law).
Student should add a unit test verifying this before training.

### Estimated Experiment Cost
Full 30-epoch run. EP5 gate: val_wss ≤ 7.0% required (if worse, GATr embedding is
mis-configured and should be debugged, not trained further).

## H_D: Charbonnier / Pseudo-Huber Supplementary WSS Loss

### Hypothesis
Add a Charbonnier loss (smooth L1 generalization) as a supplementary term alongside the
existing MSE on the WSS channels only. The Charbonnier functional is:
  `L_charb(δ) = sqrt(ε² + δ²) − ε`
where δ = pred − gt. This is differentiable everywhere (unlike L1), robust to the large
outliers in τ_z (high-magnitude separation-region WSS), and reduces gradient saturation
on the low-WSS regions where MSE over-penalizes.

The WSS field spans 1–2 orders of magnitude across the vehicle surface: stagnation regions
near zero, roof/hood boundary layer ~5–20 Pa·m⁻¹, rear wake separation ~50–150 Pa·m⁻¹.
MSE squares this already large range, causing gradient signal to be dominated by the high-WSS
peaks. Charbonnier provides a more uniform gradient across the dynamic range.

### Why This Addresses the Current Bottleneck
τ_z error (8.747% test) is concentrated in the rear separation bubble where WSS magnitude
is highest and most variable. This is where MSE gradients are largest, meaning the optimizer
chases peak values at the cost of mid-range accuracy. A supplementary Charbonnier term with
small ε reweights gradient toward mid-range WSS, which is where the bulk of the relative L2
error metric (denominator = mean |WSS|) accumulates.

Wave 27 constraint compliance: additive to MSE (never replaces it), applied only to WSS
channels. This is safe.

### Key Papers
- **Charbonnier loss for image reconstruction** (Charbonnier et al., 1994): Original.
- **UFlow: Learning to Estimate Optical Flow with Adaptive Smoothness** (Jonschkowski et al.,
  ECCV 2020): Used Charbonnier for spatial gradient loss in flow fields; analogous to WSS
  spatial variation.
- **DeepSurrogate for CFD** (Kashefi & Mukerji, 2022): Used smooth L1 on pressure fields
  with similar dynamic range argument.

### Implementation

```python
def charbonnier_loss(pred, target, eps=1e-3, reduction='mean'):
    """
    Charbonnier (pseudo-Huber) loss: sqrt(eps^2 + (pred-target)^2) - eps
    Supplementary to MSE: apply only to WSS channels.
    """
    delta = pred - target
    return (torch.sqrt(eps**2 + delta**2) - eps).mean()

# In loss computation (after existing MSE):
loss_wss_mse = F.mse_loss(pred_wss, gt_wss)
loss_wss_charb = charbonnier_loss(pred_wss, gt_wss, eps=1e-3)
# Weight: start at 0.1x MSE weight; tune if needed
loss_total = loss_existing + 0.1 * loss_wss_charb
```

CLI flag (if loss weights are configurable):
```
--wss_charbonnier_weight 0.1 \
--wss_charbonnier_eps 1e-3
```

Key hyperparameter: `eps`. Smaller eps (1e-4) → closer to L1 (robust). Larger eps (1e-2)
→ closer to MSE (smooth). Start at 1e-3 which transitions at |δ| ≈ 0.001, approximately
the noise floor of WSS predictions.

### Expected Impact
- WSS improvement: 0.1–0.3pp (primarily τ_z via dynamic range reweighting)
- vol_p / surf_p: Unaffected (WSS-only loss term)
- Risk: Low. Weight=0.1 is conservative; worst case is negligible effect.

### Implementation Difficulty: Very Low
4 lines of code. Zero dependencies. Clean isolation.

### Estimated Experiment Cost
Full 30-epoch run. Gate: EP10 val_wss must show ≥ 0.05pp improvement vs baseline to
justify continuing (otherwise the term is too small to matter and weight should be increased).
Can be composed with other changes (e.g., H_A SOAP + H_D Charbonnier) if both win
independently.

## H_E: Surface Area Channel Activation (Free Gain)

### Hypothesis
The surface input has 7 channels: `[x, y, z, nx, ny, nz, area]`. Channel 6 (`area`) is
identically ZERO across all batches in the current dataset (confirmed by tanjiro EP0 audit,
PR #1117). This means the model is effectively running with 6 channels, and the area-weighted
geometric structure (critical for WSS integration: ∫ τ dA) is completely absent from the
input. Populate `area` with the actual surface element area (triangle face area for each point
in the mesh), and retrain. No model change needed — the linear input projection already has
weight for channel 6, it just receives zero signal.

### Why This Addresses the Current Bottleneck
WSS relative L2 error is computed as `||τ_pred - τ_gt||₂ / ||τ_gt||₂`. The denominator is
area-weighted implicitly by the point distribution. Surface elements in high-curvature regions
(nose tip, rear pillar) have smaller area than flat regions (roof, hood), yet equal point
density in the current sampling scheme. The model cannot distinguish a small-area high-curvature
point from a large-area flat-panel point — both have identical geometry signal. Adding `area`
provides this critical distinguisher.

Additionally, τ_z error is concentrated in the rear separation bubble where surface element
area distribution is complex and non-uniform. Area normalization helps the model weight
its τ_z predictions appropriately by local panel size.

### Key Papers
- **DrivAerML dataset paper** (Windsor et al., 2024): Section on mesh structure and surface
  element distribution. Area is part of the formal dataset specification but may not be
  populated in the processed format.
- **FEniCS / OpenFOAM mesh structure**: Standard CFD mesh faces carry area as first-class
  attribute.

### Implementation

In the dataset preprocessing pipeline, compute triangle area for each surface point:

```python
def compute_surface_element_areas(surface_points, triangles):
    """
    surface_points: (N, 3) — x, y, z coordinates
    triangles: (T, 3) — triangle face indices
    Returns: (N,) — area contribution per point (sum of adjacent triangle areas / 3)
    """
    v0 = surface_points[triangles[:, 0]]
    v1 = surface_points[triangles[:, 1]]
    v2 = surface_points[triangles[:, 2]]
    face_areas = 0.5 * torch.norm(torch.cross(v1 - v0, v2 - v0, dim=1), dim=1)
    # Distribute to vertices (1/3 of each adjacent face)
    vertex_areas = torch.zeros(len(surface_points))
    for i in range(3):
        vertex_areas.scatter_add_(0, triangles[:, i], face_areas / 3.0)
    return vertex_areas

# Normalize to unit mean before inserting as channel 6
area_normed = vertex_areas / vertex_areas.mean()
surface_input[:, 6] = area_normed
```

If mesh connectivity is not available in the processed dataset, a simpler proxy is the
mean distance to k-nearest neighbors (approximates local area density):
```python
# kNN proxy for area (no mesh connectivity needed)
from sklearn.neighbors import NearestNeighbors
nbrs = NearestNeighbors(n_neighbors=7).fit(surface_pts)
dists, _ = nbrs.kneighbors(surface_pts)
area_proxy = dists[:, 1:].mean(axis=1)**2  # mean_dist^2 ≈ local area
area_proxy /= area_proxy.mean()
surface_input[:, 6] = area_proxy
```

NO MODEL CHANGES. NO ARCHITECTURE CHANGES. The existing channel-6 weight in the input
linear projection already exists — it just needs nonzero input.

### Expected Impact
- WSS improvement: 0.1–0.4pp (mechanism: area signal helps τ_z weighting in separation regions)
- vol_p / surf_p: Likely slightly positive (richer surface representation improves all tasks
  via shared backbone)
- Risk: Very low. Worst case: the existing channel-6 weight was regularized toward zero
  during training on zero input, so the first few epochs may show a slight dip before
  recovering. EMA should buffer this.

### Implementation Difficulty: Low
Dataset preprocessing change only. If mesh connectivity is available, use exact formula.
If not, use kNN proxy. Either way: single preprocessing script, no model changes.

### Important Note
If the mesh connectivity (triangle faces) is stored in the raw dataset at
`/mnt/new-pvc/Processed/drivaerml_processed_rawcanon_20260511`, student should use exact
area. If only point clouds are stored (no connectivity), use the kNN proxy. Student must
verify which format is available before implementing.

### Estimated Experiment Cost
Full 30-epoch run. Gate: EP5 val_wss must be ≤ 7.0%. If EP5 shows degradation
(val_wss > baseline EP5), check that area normalization is correct (should have mean=1,
std≈0.5–2.0, NOT a constant value).

## H_F: Reynolds-Number / Velocity-Magnitude Scaling Augmentation

### Hypothesis
Apply a physically valid data augmentation: multiply the inlet velocity magnitude by
λ ~ Uniform(0.7, 1.3), then scale all WSS predictions by λ² and all pressure (vol_p, surf_p)
predictions by λ². This is exact under incompressible Navier-Stokes at matched Reynolds
number scaling: U → λU implies τ_w → λ²τ_w, p → λ²p. This is mathematically exact for
laminar flow and approximate for turbulent flow (within the boundary layer assumption regime).

This augmentation PRESERVES WIND DIRECTION (Wave 27 compliant) and multiplies effective
training data by 2–4× without requiring new CFD simulations.

### Why This Addresses the Current Bottleneck
The DrivAerML dataset has 400 training cases. All training runs are at a single nominal
Reynolds number (the dataset specification uses a reference velocity). The model has never
seen slightly different velocities — it must generalize across geometry but cannot leverage
the physical scaling law that connects different velocity magnitudes. Adding λ-scaling
teaches the model that τ_z scales as λ²τ_z, which imposes a strong physical prior on the
decoder's τ_z output. This is essentially a physics-informed data augmentation that costs
zero additional compute (augmentation applied on-the-fly during training).

### Key Papers
- **Reynolds number scaling for turbulent boundary layers** (Smits et al., Ann. Rev. Fluid
  Mech. 2011): Theoretical foundation for λ² scaling law.
- **Physics-informed data augmentation for CFD** (Geneva & Zabaras, CMAME 2020):
  Symmetry-based augmentations for PDE surrogates; velocity scaling is one of the
  canonical examples.
- **DrivAerML** (Windsor et al., 2024): All cases at matched nominal conditions — no
  velocity variation in training data.

### Implementation

```python
class VelocityScalingAugmentation:
    """
    Augment training samples by scaling inlet velocity magnitude.
    WSS and pressure scale as λ^2 under incompressible NS.
    """
    def __init__(self, lambda_range=(0.7, 1.3), p_apply=0.5):
        self.lambda_range = lambda_range
        self.p_apply = p_apply

    def __call__(self, sample):
        if random.random() > self.p_apply:
            return sample
        lam = random.uniform(*self.lambda_range)
        lam2 = lam ** 2
        # Scale WSS channels (τ_x, τ_y, τ_z) by λ²
        sample['surface_y'][:, :3] *= lam2  # WSS = first 3 channels
        # Scale pressure by λ²
        sample['surface_y'][:, 3] *= lam2   # surf_p = channel 3
        sample['volume_y'][:, 0] *= lam2    # vol_p = channel 0
        # Scale velocity magnitude in surface geometry features if present
        # (position x,y,z and normals nx,ny,nz do NOT scale — geometry is fixed)
        return sample
```

Apply this augmentation in the DataLoader collation step alongside the existing Y-symmetry
augmentation (p=0.5). Total effective augmentation: 4× data (Y-sym × λ-scale combinations).

CLI flags to add:
```
--velocity_scale_aug_prob 0.5 \
--velocity_scale_lambda_min 0.7 \
--velocity_scale_lambda_max 1.3
```

### Critical Constraint
If `volume_x` contains SDF-based features or normalized coordinates, these do NOT scale
with velocity (geometry is fixed). Only output channels `surface_y` (WSS, surf_p) and
`volume_y` (vol_p) should be scaled. Do NOT scale `surface_x` or `volume_x` input features.

### Expected Impact
- WSS improvement: 0.2–0.5pp (primary mechanism: λ² consistency enforces physical scaling
  law on τ_z; model learns that high-speed → high τ_z, low-speed → low τ_z)
- vol_p / surf_p: Likely improved (same λ² scaling, more training diversity)
- Risk: If the dataset uses dimensionless normalized outputs (WSS already divided by ρU²),
  then λ scaling should NOT be applied — it would double-count. Student must verify output
  normalization before implementing. Check the `normalizer` module in the codebase.

### Implementation Difficulty: Low-Medium
Small augmentation class (10 lines). Critical: verify output normalization scheme first.
If outputs are already dimensionless (Cp / Cf style), this augmentation is invalid.

### Estimated Experiment Cost
Full 30-epoch run. Gate: EP5 val_wss ≤ 7.0%. Early signal at EP3: does val_wss move
more than 0.1pp vs baseline EP3? If not, check that the augmentation is actually being
applied (add debug log of λ values at first batch).

## H_G: Sophia Optimizer (Alternative Second-Order, Lighter Memory than SOAP)

### Hypothesis
Replace Lion with Sophia (ICLR 2024): a lightweight second-order optimizer that estimates
the diagonal Hessian periodically via Hutchinson's estimator and clips updates element-wise
by `max(h, gamma)` where h is the diagonal Hessian estimate. Sophia is computationally
cheaper than SOAP (no matrix decomposition) but provides more curvature information than
AdamW's second-moment estimate. For the τ_z prediction head where the Hessian is likely
ill-conditioned (high curvature in some directions, low in others), Sophia's element-wise
Hessian clipping prevents the optimizer from taking too-large steps on low-curvature
directions that amplify gradient noise.

### Why This Addresses the Current Bottleneck
H4 showed Lion's sign-update amplifies per-axis gradient noise. Sophia is positioned between
AdamW (H8, diagonal 2nd moment) and SOAP (H_A, full matrix preconditioner). If H8 (AdamW)
wins, Sophia is the natural next step — it adds Hessian information at modest cost. If
H_A (SOAP) wins, Sophia is a cheaper alternative worth confirming. Either way, Sophia
fills an important node in the optimizer exploration tree.

### Key Papers
- **Sophia: A Scalable Stochastic Second-Order Optimizer for Language Model Pre-training**
  (Liu et al., ICLR 2024): https://arxiv.org/abs/2305.14342 — 2× speedup vs AdamW to same
  loss; Hutchinson Hessian estimator; γ=0.01 recommended starting point.
- **sophia-optimizer** (github.com/Liuhong99/Sophia): Reference implementation, PyTorch.

### Implementation

```python
# pip install sophia-optimizer (or clone github.com/Liuhong99/Sophia)
from sophia import SophiaG

optimizer = SophiaG(
    model.parameters(),
    lr=1e-3,          # Sophia effective LR is ~2-5x Adam LR for same convergence speed
    betas=(0.965, 0.99),
    rho=0.04,         # Gradient clipping threshold (analogous to clip_grad_norm)
    weight_decay=1e-2,
    gamma=0.01,       # Hessian clipping: max(h, gamma) in denominator
    k=10,             # Estimate Hessian every k steps (cheap)
)
```

The `k=10` Hutchinson estimation means every 10th step computes a Hessian-vector product
(HVP). This adds ~10% compute overhead — acceptable for 8-GPU DDP.

CLI flags:
```

--lr 1e-3 \
--sophia_rho 0.04 \
--sophia_gamma 0.01 \
--sophia_k 10
```

### Expected Impact
- WSS improvement: 0.2–0.5pp (diagonal Hessian clips τ_z update noise)
- vol_p / surf_p: Neutral to slightly positive
- Risk: Sophia's Hutchinson estimator requires backpropagating twice per Hessian step.
  This is numerically safe but requires `retain_graph=True` in the backward call for the
  HVP step. Student must verify compatibility with GradNorm, which also requires
  double backward. Sophia's `retain_graph` usage may conflict with GradNorm's backward pass.

## H_H: Physics-Informed No-Penetration Consistency Loss

### Hypothesis
Add a supplementary loss enforcing the physical constraint that wall shear stress is
tangential to the surface (τ · n = 0, where n is the surface normal). In CFD, WSS by
definition has zero normal component — all stress is in the tangential plane. The current
MSE decoder does not enforce this constraint; the model can freely predict WSS components
that violate it. Adding this as a soft penalty on the normal component of the predicted
WSS should improve τ_z specifically: the rear separation bubble is where the surface normal
has a large z-component (upswept surface), and the model may be predicting spurious
τ_z via the normal direction.

The constraint: `L_perp = mean(|(pred_wss · n)|²)` where n is the unit surface normal.
This is supplementary to MSE (Wave 27 compliant) and computationally trivial.

### Why This Addresses the Current Bottleneck
τ_z (8.747% test) has the highest error among WSS components. For the rear of the vehicle
(trunk lid, C-pillar, spoiler), the surface normal has a significant z-component — pointing
partially upward and rearward. The MSE decoder can "cheat" by predicting τ_z via the normal
component, which does not contribute to the physical WSS but looks similar in the local
coordinate frame. Enforcing τ · n = 0 closes this shortcut and forces the model to predict
genuinely tangential shear stress.

The surface normals (nx, ny, nz) are already in the input (channels 3–5 of surface_x).
This loss costs one dot product per surface point.

### Key Papers
- **Physics-Informed Neural Networks (PINNs)** (Raissi et al., JCP 2019): No-penetration
  boundary condition enforcement as soft penalty.
- **Boundary condition enforcement in neural PDE solvers** (Lu et al., Nature Machine
  Intelligence 2021): Hard vs soft enforcement; soft penalty found effective for complex
  geometries.
- **CFD boundary layer theory** (White, Viscous Fluid Flow): τ · n = 0 is an exact
  kinematic constraint for incompressible attached flow.

### Implementation

```python
def no_penetration_loss(pred_wss, surface_normals):
    """
    Enforce τ · n = 0 (WSS is tangential to surface).
    pred_wss: (B, N_surf, 3) — predicted τ_x, τ_y, τ_z
    surface_normals: (B, N_surf, 3) — unit normals nx, ny, nz (from surface_x channels 3-5)
    Returns: scalar loss
    """
    # Dot product of predicted WSS with surface normal
    normal_component = (pred_wss * surface_normals).sum(dim=-1)  # (B, N_surf)
    return (normal_component ** 2).mean()

# In loss computation:
# Extract normals from input (channels 3, 4, 5 of surface_x)
normals = surface_x[:, :, 3:6]  # (B, N_surf, 3) — already unit-normalized
loss_perp = no_penetration_loss(pred_wss_denormed, normals)
loss_total = loss_existing + 0.01 * loss_perp  # Start at 1% weight
```

Important: apply this loss AFTER denormalization (or use normalized normals with
normalized WSS — the constraint is linear so it holds in either space as long as the
same normalization is applied to both).

Weight tuning: start at 0.01. If τ_z shows no movement at EP5, increase to 0.05. If
vol_p or surf_p degrade, reduce to 0.005. The constraint should be soft — the model's
existing gradient signal from MSE should dominate.

### Expected Impact
- WSS improvement: 0.1–0.3pp (primarily τ_z via normal-component elimination)
- vol_p / surf_p: Unaffected (surface-only loss)
- Risk: If the dataset WSS is already in global coordinates with normals also in global
  coordinates, the constraint is correctly applied. If WSS is in local (surface-tangent)
  coordinates, then τ · n ≡ 0 by construction and this loss is trivially zero — no harm
  but no benefit. Student must verify coordinate system before implementing.

### Implementation Difficulty: Very Low
5 lines of code. No dependencies. Verify coordinate system first.

### Estimated Experiment Cost
Full 30-epoch run. Gate: EP5 val_wss must improve by ≥ 0.05pp vs baseline. If not,
check that the normal-component loss is actually nonzero (debug: print
`loss_perp.item()` at first step — should be > 0 if WSS is in global coordinates).

## Priority Ordering and Experiment Tree

```
TIER 1 (Plateau Protocol — Optimizer Regime Change):
  H_A: SOAP optimizer                    [HIGHEST PRIORITY — deepest second-order info]
  H_G: Sophia optimizer                  [Alternative if SOAP memory fails]

TIER 2 (Data / Training Curriculum):
  H_B: Coarse-to-fine curriculum         [Explicitly plateau-breaking, arXiv:2509.13138]
  H_E: Surface area channel activation   [Free gain — no model change, no architecture risk]
  H_F: Velocity-magnitude augmentation   [Physics-valid, 2-4x data for free]

TIER 3 (Loss Formulation):
  H_D: Charbonnier supplementary loss    [Safe, additive, handles WSS dynamic range]
  H_H: No-penetration consistency loss   [Physical constraint, 5 lines of code]

TIER 4 (Architecture):
  H_C: GATr equivariant decoder          [Biggest potential upside, highest implementation cost]
```

Decision tree:
```
IF H_A (SOAP) wins AND H_B (curriculum) wins INDEPENDENTLY:
  → Try H_A + H_B composed (different optimizers are orthogonal to curriculum)
IF H_A fails (SOAP memory issue):
  → Fall back to H_G (Sophia)
IF H_A wins BUT vol_p drifts up:
  → Compose H_A + H_D (Charbonnier adds surface-task gradient to stabilize vol_p)
IF H_E (area channel) wins:
  → Always compose with future winners (it's a dataset fix, orthogonal to all else)
IF H_F (velocity scaling) wins:
  → Verify λ² scaling is correct (debug: check that training loss decreases normally)
IF all TIER 1-3 fail:
  → H_C (GATr decoder) is the bold architecture swing — commit a full 30-epoch run
IF H_C wins on val_wss but vol_p breaches floor:
  → Add surface_loss_weight=1.5 (H7 mechanism) to stabilize vol_p via surface task boost
```

Stop condition: If val_wss fails to improve below 6.7% (current SOTA level) across all
8 proposed ideas, the bottleneck is likely in the backbone architecture (Transolver's
attention mechanism cannot represent τ_z sufficiently), and the research program should
pivot to full backbone replacement (FNO, Factformer, or full GATr).
